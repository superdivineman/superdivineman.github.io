1. Why Fine-Tune Speech Models?

Pre-trained speech models are trained on massive, diverse audio datasets — they know general English speech patterns but often struggle with:

  • Domain-specific vocabulary — medical notes, legal testimony, technical documentation, code
  • Accented speech — non-native speakers, regional accents
  • Noisy environments — factories, cars, call centers
  • Low-resource languages — adapting Whisper to dialects or minority languages
  • Specialized formats — timestamps, punctuation, speaker diarization

Fine-tuning a pre-trained model on 1-10 hours of domain-specific data can dramatically improve WER (Word Error Rate) on your target task.

Rule of Thumb
If your target domain has specific vocabulary or acoustic conditions, fine-tuning is almost always worth it. 10 hours of labeled domain data + a good pre-trained model beats 1,000 hours of generic data.

3.1 Audio Format Requirements

# Supported formats: WAV (preferred), FLAC, MP3, OGG, M4A
# Recommended: 16-bit PCM WAV, 16kHz mono

# Convert any audio to 16kHz mono WAV
from pydub import AudioSegment

def convert_audio(path, output_path, target_sr=16000):
    audio = AudioSegment.from_file(path)
    audio = audio.set_frame_rate(target_sr).set_channels(1).set_sample_width(2)
    audio.export(output_path, format="wav")

# Batch convert
import glob, pathlib

3.3 Text Normalization

# Speech datasets need consistent text normalization
import re

def normalize_text(text):
    text = text.lower()
    text = re.sub(r"[^\w\s]", "", text)   # remove punctuation
    text = re.sub(r"\s+", " ", text)        # collapse whitespace
    text = text.strip()
    return text

# Whisper has built-in normalization: `normalizer` from transformers
from transformers import WhisperTokenizer, WhisperNormalizer

4. Fine-Tuning Whisper (OpenAI)

4.1 Why Whisper?

Whisper is an encoder-decoder Transformer. The encoder processes audio into hidden states; the decoder generates text autoregressively. Fine-tuning replaces the decoder head and optionally unfreezes parts of the encoder for domain-specific acoustic patterns.

4.3 Full Fine-Tuning Code

import torch
from transformers import (
    WhisperProcessor, WhisperForConditionalGeneration,
    WhisperTokenizer, WhisperNormalizer,
    TrainingArguments, Trainer, DataCollatorWithPadding
)
from datasets import load_dataset, Audio

# ── Load pre-trained model ───────────────────────────────────────────────────
model_name = "openai/whisper-small"   # 244M params. Also: tiny(39M), base(74M), small(244M), medium(769M), large(1550M)
processor = WhisperProcessor.from_pretrained(model_name)
model = WhisperForConditionalGeneration.from_pretrained(model_name)
tokenizer = WhisperTokenizer.from_pretrained(model_name)
normalizer = WhisperNormalizer()

# ── Force English transcription (if you only need English) ───────────────────
model.config.forced_decoder_ids = None
model.config.suppress_tokens = []

# ── Prepare dataset ───────────────────────────────────────────────────────────
# Assumes train_manifest.json in {"audio_path": ..., "text": ...} format
from datasets import Dataset

def load_manifest(path):
    import json
    with open(path) as f:
        items = [json.loads(line) for line in f]
    return items

def prepare_dataset(item):
    audio_path = item["audio_path"]
    text = normalizer(item["text"])

    # Load + resample audio
    audio = item["audio"]
    audio_array = audio["array"]
    sampling_rate = audio["sampling_rate"]

    # Process audio → features
    input_features = processor(
        audio_array, sampling_rate=sampling_rate,
        return_tensors="pt"
    ).input_features[0]

    # Tokenize text → labels
    labels = tokenizer(text, truncation=True, max_length=448).input_ids

    item["input_features"] = input_features
    item["labels"] = labels
    return item

# Load dataset
raw_dataset = load_dataset("json", data_files={"train": "./train_manifest.json"}, split="train")
raw_dataset = raw_dataset.cast_column("audio", Audio(sampling_rate=16000))
dataset = raw_dataset.map(prepare_dataset, remove_columns=raw_dataset.column_names)

# ── Data collator ( Whisper requires special handling ) ──────────────────────
class WhisperDataCollator:
    def __init__(self, processor):
        self.processor = processor
        self.processor.tokenizer.pad_token = self.processor.tokenizer.eos_token

    def __call__(self, features):
        # Pad input_features (variable-length audio)
        input_features = [
            {"input_features": f["input_features"]} for f in features
        ]
        batch = self.processor.feature_augmentor.pad(input_features, return_tensors="pt")

        # Pad labels
        label_features = [
            {"input_ids": f["labels"]} for f in features
        ]
        labels_batch = self.processor.tokenizer.pad(label_features, return_tensors="pt")
        labels = labels_batch["input_ids"].masked_fill(labels_batch.attention_mask.ne(1), -100)

        batch["labels"] = labels
        return batch

data_collator = WhisperDataCollator(processor)

# ── Training arguments ──────────────────────────────────────────────────────
training_args = TrainingArguments(
    output_dir="./whisper-finetuned",
    per_device_train_batch_size=8,
    gradient_accumulation_steps=2,          # effective batch = 16
    learning_rate=1e-5,                      # Whisper is already well-trained — use low LR
    warmup_steps=100,
    num_train_epochs=3,
    fp16=True,                              # use bf16 on Ampere+
    logging_steps=25,
    save_strategy="epoch",
    report_to="tensorboard",
    max_grad_norm=1.0,                       # gradient clipping
)

# ── Trainer ──────────────────────────────────────────────────────────────────
trainer = Trainer(
    model=model,
    args=training_args,
    train_dataset=dataset,
    data_collator=data_collator,
)
trainer.train()
model.save_pretrained("./whisper-finetuned/final")

5. Fine-Tuning Wav2Vec 2.0

Wav2Vec 2.0 uses a contrastive learning approach: the encoder processes raw audio into latent representations, and a quantization module provides training targets. Fine-tuning replaces the quantizer with a labeled classification head.

from transformers import (
    Wav2Vec2Processor, Wav2Vec2ForCTC,
    Wav2Vec2FeatureExtractor, Wav2Vec2CTCTokenizer
)
from datasets import load_dataset, Audio

# ── Load base model ───────────────────────────────────────────────────────────
model_name = "facebook/wav2vec2-base-960h"   # 960h = trained on 960 hours of LibriSpeech
processor = Wav2Vec2Processor.from_pretrained(model_name)
model = Wav2Vec2ForCTC.from_pretrained(model_name)

# Freeze most layers — fine-tune only the last few Transformer blocks + classifier
for param in model.wav2vec2.parameters():
    param.requires_grad = False
for param in model.wav2vec2.encoder.layers[-4:].parameters():  # unfreeze last 4 encoder layers
    param.requires_grad = True
model.lm_head.requires_grad = True

# ── Resample all audio to 16kHz ──────────────────────────────────────────────
dataset = load_dataset("path/to/your/manifest", split="train")
dataset = dataset.cast_column("audio", Audio(sampling_rate=16000))

# ── Preprocess ───────────────────────────────────────────────────────────────
def prepare_batch(batch):
    audio = batch["audio"]
    batch["input_values"] = processor(audio["array"], sampling_rate=16000).input_values[0]
    with processor.as_target_processor():
        batch["labels"] = processor(batch["text"]).input_ids
    return batch

dataset = dataset.map(prepare_batch, remove_columns=dataset.column_names)

# ── Data collator ─────────────────────────────────────────────────────────────
class Wav2Vec2Collator:
    def __call__(self, features):
        input_values = [{"input_values": f["input_values"]} for f in features]
        batch = processor.pad(input_values, return_tensors="pt", padding=True)
        labels = [{"labels": f["labels"]} for f in features]
        label_batch = processor.tokenizer.pad(labels, return_tensors="pt")
        batch["labels"] = label_batch["input_ids"].masked_fill(
            label_batch.attention_mask.ne(1), -100
        )
        return batch

# ── Training ──────────────────────────────────────────────────────────────────
# Use HuggingFace Trainer with CTC loss (Wav2Vec2 uses CTC, not cross-entropy)
from transformers import TrainingArguments, Trainer

training_args = TrainingArguments(
    output_dir="./wav2vec2-finetuned",
    group_by_length=True,                    # group similar-length samples for efficiency
    per_device_train_batch_size=8,
    gradient_accumulation_steps=4,
    learning_rate=1e-4,
    warmup_steps=100,
    num_train_epochs=10,
    fp16=True,
    logging_steps=50,
)
trainer = Trainer(
    model=model, args=training_args,
    train_dataset=dataset,
    data_collator=Wav2Vec2Collator(),
)
trainer.train()

6. Coqui STT: Transfer Learning

Coqui STT (formerly Mozilla STT) is designed specifically for fine-tuning on custom datasets. It uses transfer learning from a pre-trained checkpoint and is optimized for on-device inference.

# Install Coqui STT

# Download pre-trained model
# Available: quartznet15x5 (93M), quartznet15x5se (95M), citrinet-1024 (321M)
coqui-stt-train \
  --initialize_from_checkpoint model_checkpoint.pbmm \
  --n_hidden 2048 \
  --train_files train.csv \
  --dev_files dev.csv \
  --test_files test.csv \
  --sample_rate 16000 \
  --epochs 50 \
  --learning_rate 0.0003 \
  --export_tflite \
  --export_dir ./coqui-output
Coqui vs HuggingFace
Coqui STT is better for production on embedded devices (Raspberry Pi, mobile). HuggingFace models (Whisper, Wav2Vec) are better for cloud deployment and research flexibility. For fine-tuning: Whisper/Wav2Vec give better raw quality; Coqui gives better on-device performance.

7. Data Augmentation for Speech

Speech data is often limited. Augmentation artificially expands your dataset:

import torch
import torchaudio
import torchaudio.transforms as T

# ── Speed & Pitch perturbation ───────────────────────────────────────────────
# Changes prosody without changing content — very effective for robustness# Changes prosody without changing content — very effective for robustness
def speed_augment(waveform, sample_rate):
    speed_factor = torch.randint(90, 110, [1]).item() / 100.0
    sox_effects = [
        ["speed", str(speed_factor)],
        ["rate", str(sample_rate)],
    ]
    augmented, _ = torchaudio.sox_effects.apply_effects_tensor(waveform, sample_rate, sox_effects)
    return augmented

# ── Noise injection ──────────────────────────────────────────────────────────
def add_noise(waveform, noise_factor=0.005):
    noise = torch.randn_like(waveform) * noise_factor
    return waveform + noise

# ── SpecAugment (frequency masking) ──────────────────────────────────────────
# Randomly mask frequency bands in the spectrogram — proven effective in speech tasks
spec_augment = T.TimeStretch(hop_length=256).cuda()

# ── Background noise from external files ─────────────────────────────────────
def mix_background_noise(waveform, noise_dir, snr_db=10):
    import glob, random, torchaudio
    noise_files = glob.glob(f"{noise_dir}/*.wav")
    noise_wav, sr = torchaudio.load(random.choice(noise_files))
    noise_wav = noise_wav.mean(0)[:len(waveform)]
    # SNR: signal-to-noise ratio in dB
    snr = 10 ** (snr_db / 20)
    signal_power = (waveform ** 2).mean()
    noise_power = (noise_wav ** 2).mean()
    scale = (signal_power / noise_power / snr) ** 0.5
    return waveform + noise_wav * scale

8. Evaluation Metrics

The standard metric for ASR is Word Error Rate (WER). Lower is better.

pip install jiwer

from jiwer import wer, cer

# WER: word-level errors (substitutions + deletions + insertions / total reference words)
# CER: character-level errors — useful for languages without word boundaries
wer_score = wer(reference, hypothesis)
cer_score = cer(reference, hypothesis)
print(f"WER: {wer_score:.2%}  |  CER: {cer_score:.2%}")

# Evaluate on test set
def evaluate_model(model, test_dataset, processor):
    model.eval()
    predictions, references = [], []
    for item in test_dataset:
        input_features = processor.feature_extractor(
            item["audio"]["array"], sampling_rate=16000, return_tensors="pt"
        ).input_features
        with torch.no_grad():
            generated_ids = model.generate(input_features)
        transcript = processor.batch_decode(generated_ids)[0]
        predictions.append(transcript)
        references.append(item["text"])
    return wer(references, predictions)
WER RangeQualityNotes
< 5%ExcellentNearly perfect transcription
5–10%GoodMinor errors, usable for most applications
10–20%FairErrors noticeable; may need post-editing
> 20%PoorNeeds significant improvement or more data

9. Deployment

Whisper (HuggingFace + FastAPI)

pip install fastapi uvicorn transformers torch

# server.py
from fastapi import FastAPI, UploadFile, File
from transformers import pipeline
import torch

app = FastAPI()
asr = pipeline("automatic-speech-recognition", model="./whisper-finetuned", device=0)

@app.post("/transcribe")
async def transcribe(file: UploadFile = File(...)):
    audio_bytes = await file.read()
    # Save to temp file (or use BytesIO)
    import tempfile, os
    with tempfile.NamedTemporaryFile(suffix=".wav", delete=False) as tmp:
        tmp.write(audio_bytes)
        result = asr(tmp.name)
        os.unlink(tmp.name)
    return result

10. Checklist

  • ☐ Audio format: 16kHz mono WAV (or convert with pydub)
  • ☐ Text normalization: lowercase, remove punctuation, strip whitespace
  • ☐ Dataset split: 80/10/10 train/val/test or 90/10 train/test
  • ☐ WER baseline: evaluate pre-trained model on your test set BEFORE fine-tuning
  • ☐ Learning rate: 1e-5 for Whisper (already well-trained), 1e-4 for Wav2Vec
  • ☐ Epochs: 3-10 depending on dataset size; monitor val WER to avoid overfitting
  • ☐ Augmentation: speed/pitch perturbation + noise injection
  • ☐ Eval: report WER + CER on held-out test set
  • ☐ Save strategy: save best model by val WER, not last epoch
  • ☐ Production: export to ONNX or use FastAPI + GPU for serving
